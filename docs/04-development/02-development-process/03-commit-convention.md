# Commit Convention

## Tổng quan

Dự án Insurance System tuân thủ [Conventional Commits](https://www.conventionalcommits.org/) specification để đảm bảo commit history rõ ràng, dễ đọc và hỗ trợ auto-generate changelog.

---

## Format

```
<type>(<scope>): <subject>

[optional body]

[optional footer(s)]
```

### Ví dụ

```
feat(policy): add premium calculation for life insurance

Implement premium calculation engine with age-based risk factors.
Supports standard and premium plans with different coverage levels.

Closes #123
```

---

## Types

| Type | Mô tả | Ví dụ |
|------|--------|-------|
| `feat` | Tính năng mới | `feat(auth): add JWT refresh token` |
| `fix` | Bug fix | `fix(payment): resolve double charge issue` |
| `docs` | Documentation | `docs(api): update endpoint descriptions` |
| `style` | Formatting (không ảnh hưởng logic) | `style: fix indentation in service files` |
| `refactor` | Refactoring (không thêm feature/fix bug) | `refactor(policy): extract validation logic` |
| `perf` | Performance improvement | `perf(search): add database indexing` |
| `test` | Thêm/sửa tests | `test(claims): add unit tests for submission` |
| `build` | Build system changes | `build: update TypeScript to 5.4` |
| `ci` | CI/CD changes | `ci: add staging deployment workflow` |
| `chore` | Maintenance tasks | `chore: update dependencies` |
| `revert` | Revert commit trước | `revert: revert "feat(auth): add OAuth"` |

## Subject Rules

### DO ✅

- Viết bằng tiếng Anh
- Sử dụng **imperative mood** (add, fix, update - không phải added, fixed, updated)
- Bắt đầu bằng **lowercase**
- Không kết thúc bằng dấu chấm (`.`)
- Giới hạn **50 ký tự**
- Mô tả **what**, không phải **how**

### DON'T ❌

```bash
# ❌ Past tense
feat(policy): added search functionality

# ❌ Capitalized
feat(policy): Add search functionality

# ❌ Ends with period
feat(policy): add search functionality.

# ❌ Too vague
fix: fix bug
feat: update code
chore: changes

# ❌ Too long
feat(policy): add comprehensive search functionality with multiple filter options including date range type and status
```

### Ví dụ tốt ✅

```bash
feat(policy): add search with advanced filters
fix(auth): resolve token expiration race condition
docs(api): document rate limiting headers
refactor(premium): extract risk calculation to separate service
test(claims): add integration tests for approval flow
perf(search): implement query result caching
```

## Body (Optional)

### Khi nào cần Body?

- Thay đổi phức tạp cần giải thích
- Breaking changes
- Cần context về **why** (không chỉ what)
- Multiple related changes trong 1 commit

### Format

```
feat(premium): redesign calculation engine

WHAT: Replace flat-rate calculation with risk-based model.

WHY: Previous model didn't account for age bands and medical
history, leading to under-pricing for high-risk customers.

HOW: 
- Implement age-band risk factors (18-30, 31-45, 46-60, 60+)
- Add medical history weight multiplier
- Apply regional adjustment coefficients
```

### Rules

- Cách subject bằng 1 dòng trống
- Wrap text ở **72 ký tự**
- Giải thích **motivation** cho thay đổi
- Có thể dùng bullet points

---

## Footer (Optional)

### Breaking Changes

```
feat(api): change policy response format

BREAKING CHANGE: Policy API response now returns 'policyNumber'
instead of 'policy_number'. All API consumers need to update
their integration.

Migration: Replace all references of 'policy_number' with
'policyNumber' in API response handling.
```

### Issue References

```bash
# Close issue
Closes #123

# Reference multiple issues
Closes #123, #456
Refs #789

# External references
JIRA: INS-123
```

## Special Cases

### Merge Commits

```bash
# Auto-generated by Git, không cần modify
Merge branch 'feature/add-policy-search' into develop
```

### Revert Commits

```bash
revert: feat(auth): add OAuth2 login

This reverts commit abc1234.

Reason: OAuth provider has rate limiting issues in production.
Will re-implement with different provider.
```

### Multi-package Changes

```bash
# Khi thay đổi cả BE và FE
feat(policy): add policy search

- BE: Add search API endpoint with pagination
- FE: Add search UI with filter panel
```

### WIP Commits (trên feature branch)

```bash
# OK trên feature branch, sẽ squash trước merge
wip: policy search - API endpoint done
wip: policy search - add filter logic
wip: policy search - UI implementation

# Squash thành 1 commit khi merge:
feat(policy): add policy search with advanced filters
```

## Commit Message Examples

### Feature Development

```bash
feat(customer): add customer profile management

- Create CRUD endpoints for customer profiles
- Add avatar upload with image resize
- Implement email verification flow

Closes #45
```

### Bug Fix

```bash
fix(payment): resolve duplicate transaction on timeout

When payment gateway times out, the retry mechanism was not
checking for existing pending transactions, causing duplicate
charges.

Added idempotency key check before creating new transaction.

Closes #78
```

### Performance

```bash
perf(search): optimize policy search query

Reduce search response time from 2.5s to 200ms by:
- Adding composite index on (customer_id, status, created_at)
- Implementing cursor-based pagination
- Adding Redis cache for frequent queries

Refs #92
```

## Tooling

### Commitlint Configuration

```javascript
// commitlint.config.js
module.exports = {
  extends: ['@commitlint/config-conventional'],
  rules: {
    'type-enum': [2, 'always', [
      'feat', 'fix', 'docs', 'style', 'refactor',
      'perf', 'test', 'build', 'ci', 'chore', 'revert',
    ]],
    'scope-enum': [1, 'always', [
      'auth', 'policy', 'claims', 'payment', 'customer',
      'premium', 'notification', 'report', 'api', 'db',
      'ui', 'form', 'page', 'store', 'hook', 'route',
      'deps', 'docker', 'deploy', 'config', 'lint', 'test',
    ]],
    'subject-max-length': [2, 'always', 50],
    'body-max-line-length': [2, 'always', 72],
  },
};
```

### Git Hook Setup

```bash
# Tự động validate commit message
npx husky add .husky/commit-msg 'npx commitlint --edit $1'
```
